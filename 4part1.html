Ethereum – Extremely Easy & Detailed Notes

This document explains **Ethereum from zero**, slowly and clearly, without any fancy formatting. Pure exam-friendly notes.

1. What is Ethereum?

Ethereum is a **decentralized programmable blockchain**. Unlike Bitcoin (which mainly transfers money), Ethereum allows running programs called **smart contracts**.

  • Uses **Ether (ETH)** as its currency.
  • Smart contracts run on thousands of computers globally.
  • Provides a trustless way to execute applications.
Simple meaning: Ethereum = global computer + money + apps running without central control.

2. The Yellow Paper (Formal Definition of Ethereum)

The Yellow Paper is a technical document written by Gavin Wood. It defines:

  • How blocks are structured
  • How Ethereum updates its state
  • How smart contracts run in the Ethereum Virtual Machine (EVM)
  • How gas fees are calculated

It covers Ethereum mathematically so all implementations behave identically.

Important Concepts from Yellow Paper

  • World State: All account balances + contract storage.
  • Account Object: nonce, balance, codeHash, storageRoot.
  • State Transition Function (Π): Defines how state changes after each transaction.

3. Ethereum from a User’s Perspective

From a normal user or developer’s perspective, Ethereum works like this:

  1. User has a **wallet** (Metamask, Trust Wallet, Ledger).
  2. User sends a **transaction** (send ETH or call a smart contract).
  3. They pay **gas fee**.
  4. Validators include the transaction into a block.
  5. The network updates Ethereum’s global state.

Example:

If you transfer 1 ETH to a friend:

  1. You sign the transaction with your private key.
  2. Validators check your signature + balance.
  3. State updates: your balance –1 ETH, friend +1 ETH.

4. Ethereum Network Types

A. Mainnet

The real Ethereum blockchain. ETH has real value.

B. Testnets

Used for testing, with free ETH:

  • Goerli – major testnet
  • Sepolia – fast, lightweight testnet

C. Private Networks

Run inside organizations. No real ETH needed. Used for experiments.

5. Components of the Ethereum Ecosystem

  • Clients: Geth, Nethermind (software running Ethereum protocol)
  • Wallets: MetaMask, hardware wallets
  • Smart Contracts: Code stored and executed on Ethereum
  • DApps: Apps that use Ethereum smart contracts
  • Oracles: Provide external data (weather, price feeds)
  • Layer 2: Rollups to improve speed and reduce fees

6. Keys & Addresses

Private Key

A secret number. Whoever knows it controls your account.

Public Key

Derived from private key.

Address

Last 20 bytes of Keccak256(public key). Example: 0xA3b...d91

Address = Public identity
Private key = Password to spend ETH

7. Accounts in Ethereum

A. Externally Owned Accounts (EOA)

  • Controlled by a private key
  • Can sign transactions
  • No smart contract code

B. Contract Accounts

  • Contain smart contract code
  • Cannot send transactions themselves
  • Triggered by EOA transactions

8. Transactions

A transaction is a signed message that changes the state of Ethereum.

Fields inside a transaction:

  • nonce
  • gasLimit
  • maxFeePerGas / gasPrice
  • to (address)
  • value (ETH)
  • data (smart contract function call)
  • signature (v,r,s)

Types of Transactions

  • Send ETH
  • Deploy a smart contract
  • Call a smart contract function

9. Messages in Ethereum (Internal Calls)

Messages are not signed. They are created by smart contracts when:

  • Contract A calls Contract B
  • Contract A sends ETH to Contract B

These happen inside a transaction.

10. EVM (Ethereum Virtual Machine)

The EVM executes smart contracts.

  • Stack-based machine
  • Memory for temporary data
  • Storage for permanent contract data
  • Instructions called opcodes (e.g. ADD, MUL, CALL, SSTORE)

11. Gas (Important)

Gas measures how much computation a transaction needs.

  • Every instruction costs gas
  • If gas runs out → transaction fails
  • Unused gas is refunded
Gas fee = gasUsed × gasPrice

12. Blocks & Finality

Blocks contain:

  • block number
  • timestamp
  • transactions
  • stateRoot

Finality means a block cannot be reversed. In Ethereum PoS, finality happens after validator confirmations.

13. Validators & Staking

Validators:

  • Stake 32 ETH
  • Propose blocks
  • Vote on blocks
  • Can be slashed for cheating

14. Logs & Events

Smart contracts can emit logs. These are used for:

  • Tracking transfers
  • Front-end dapp updates

15. Summary

  • Ethereum = programmable blockchain
  • Yellow Paper = protocol rules
  • Users interact via wallets & transactions
  • Mainnet/Testnet/Private networks exist
  • Accounts = EOA + Contract accounts
  • EVM runs smart contract code
  • Gas prevents spam and rewards validators
  • Validators secure Ethereum through staking
